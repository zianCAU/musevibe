# 随身乐器「缪斯 MUSE」· 产品设计规格

**日期：** 2026-04-21  
**硬件：** Otto-Robot ESP32-S3（摄像头版）  
**项目分支：** factory  

---

## 一、产品概述

面向音乐零基础用户的随身智能乐器卡片。用户按下电容触摸按钮即可弹奏吉他和弦音效；内置「缪斯」AI 角色，支持日常音乐创作对话，并能基于摄像头拍摄的景色实时生成歌词+和弦谱，用户可以直接看屏幕演奏、跟唱。

---

## 二、硬件与针脚

### 2.1 板载资源

| 资源 | 型号/配置 |
|---|---|
| 主控 | ESP32-S3 |
| 显示屏 | ST7789 240×240 SPI LCD |
| 音频 | I2S 全双工（喇叭 + 麦克风） |
| 摄像头 | DVP 接口（已有 EspVideo 驱动） |
| 电源 | 锂电池 + 充电管理 |

### 2.2 触摸按钮接线（重要！）

Otto-Robot 摄像头版有 6 个舵机插针，但 ESP32-S3 的电容触摸外设只接通了 GPIO 1–14。  
其中舵机针脚只有 **GPIO 4、5、6、7** 在触摸范围内，GPIO 43/44 是 UART0 TX/RX，芯片内部无触摸电路，无法使用。

> **后期扩展**：若想要 6 个触摸按钮，可在 GPIO 43/44 上挂 MPR121 I2C 触摸芯片（¥5–10）。当前阶段 4 个和弦已覆盖 90%+ 流行歌曲。

#### 接线表

| 按钮编号 | 和弦 | GPIO | 触摸通道 | 原舵机位置 |
|---|---|---|---|---|
| BTN1 | **C 大三** | GPIO 4 | TOUCH4 | left_hand |
| BTN2 | **Am 小三** | GPIO 5 | TOUCH5 | left_leg |
| BTN3 | **F 大三** | GPIO 6 | TOUCH6 | left_foot |
| BTN4 | **G 大三** | GPIO 7 | TOUCH7 | right_hand |
| ~~GPIO 43~~ | — | 不可用 | UART TX | right_leg |
| ~~GPIO 44~~ | — | 不可用 | UART RX | right_foot |

#### 实物接线方法

```
每个舵机插针有 3 根针：GND / +5V / Signal
只需引出 Signal 针（靠板边那根）：

Signal 针 ──导线──> 铜箔触摸片（直径 10–15mm）

· ESP32-S3 内部自带上拉，无需外部电阻
· 导线长度建议不超过 20cm（过长会降低灵敏度）
· 铜箔/铜片/金属螺帽均可作为触摸面
```

#### 实物接线方法

```
舵机插针（3针：GND / +5V / Signal）
只需接 Signal 针（最靠近板边缘那根）即可。

Button ──── 一根导线焊到 Signal 针 ─── GPIO(4/5/6/7)
                                          │
                              ESP32-S3 内部上拉，无需外部电阻
                              触摸面：铜箔/铜片/金属贴片均可
```

#### 推荐触摸面材料

- 直径 10–15mm 的铜箔圆片（PCB 贴纸）
- 或金属螺帽/螺钉头焊上导线
- 导线长度不超过 20cm（过长会影响灵敏度）

## 三、UI 设计规格

### 3.1 视觉风格：暗夜霓虹

| 元素 | 颜色/效果 |
|---|---|
| 背景 | 深紫星空径向渐变 `#060610 → #150a2e` |
| 主题色 | 紫色霓虹 `#9c5fff` / `#a855f7` |
| 文字 | 白 / 淡紫 `#d0adff` |
| 和弦按钮 | 深紫渐变 + 紫色边框光晕，按下时高亮爆光 |
| AI 头像 | 音符卡通形象，带呼吸发光动效（3s 循环） |

### 3.2 界面①：待机/演奏模式（主界面）

布局：**左角色 + 右对话**

```
┌─────────────────────────────────────────────┐
│  ┌──────┐  │  AI对话消息流（滚动）           │  240px
│  │  🎵  │  │  ─────────────────────────────  │
│  │ MUSE │  │  [AI气泡] 来弹一段吧～          │
│  └──────┘  │  [用户气泡] 帮我写首歌          │
│            │  [AI气泡] 好的，我看到了...      │
│  88px宽   │  ─────────────────────────────  │
│            │  [ C ] [ Am] [ F ] [ G ]        │  3×2网格
└─────────────────────────────────────────────┘
                                              240px
```

**左栏（88px）：**
- AI 头像圆形（56×56px），紫色光晕呼吸动效
- 名字标签「MUSE」（8px，全大写，紫色）
- 背景：深紫半透明

**右栏（152px）：**
- 上部：对话气泡列表，AI 左对齐，用户右对齐
- 下部：4 个和弦按钮网格（2×2），标注 C / Am / F / G
- 按钮高度 24px，触按时紫色爆光 + 播放音效

**状态切换：**
- 平时：右侧显示对话 + 和弦按钮
- AI 正在生成歌词时：右侧显示进度动画「正在创作...」
- 生成完成：自动切换到界面②

### 3.3 界面②：歌词弹唱模式

布局：**逐行滚动式**，全屏覆盖主界面。

```
┌─────────────────────────────────────────────┐
│ ♪ 同桌的你·景色填词              2 / 4段  │  标题栏 28px
├─────────────────────────────────────────────┤
│ [C ] 那片山坡住着谁的故事         ← 暗淡  │
│ [Am] 秋风把叶子卷成信纸           ← 暗淡  │
│ ━━[F ]━ 你说过窗外的光是温柔的 ←当前行高亮│  紫色背景+左侧竖线
│ [G ] 我还记得那一天的云                   │
│ [Am] 也许青春就是一场风景                 │
│ [F ] 带走了却带不走你笑容                 │
├─────────────────────────────────────────────┤
│    C     Am   [F]    G              │  底部和弦条 36px
└─────────────────────────────────────────────┘
```

**逻辑：**
- 每行 = 和弦标签（26px 宽紫色小胶囊）+ 歌词文字（11px）
- 当前行：紫色背景高亮 + 左侧 2px 竖线标记
- 上方已过行：opacity 0.3 暗淡
- 底部和弦条：当前行用的和弦自动高亮发光
- 上下滑动或物理按钮翻页（按住 BTN1 上翻，BTN4 下翻）

## 四、功能规格

### 4.1 触摸演奏

1. `touch_pad` 驱动轮询 GPIO 4/5/6/7，检测触摸事件
2. 触摸确认 → 从 Flash 加载对应和弦的 PCM 音频数据
3. 通过 `AudioCodec::OutputData()` 输出
4. LVGL 界面对应按钮触发霓虹高亮动效（150ms）

**音色方案：**
- 吉他分解弦（默认）：每根弦间隔 80ms 顺序播放
- 扫弦：一次性混合输出，更响亮
- 通过语音指令切换：「换成扫弦」/「换成分解」

**和弦音频预算（Flash）：**

| 和弦 | 分解弦 | 扫弦 | 合计 |
|---|---|---|---|
| C | ~18KB | ~12KB | ~30KB |
| Am | ~18KB | ~12KB | ~30KB |
| F | ~18KB | ~12KB | ~30KB |
| G | ~18KB | ~12KB | ~30KB |
| **总计** | | | **~120KB** |

> 采用 16kHz / 16bit mono PCM，与现有 I2S 配置一致，无需重采样。

### 4.2 AI 景色填词流程

```
用户语音指令
     │
     ▼
Application 识别意图 → "创作"关键词
     │
     ▼
拍摄一帧（EspVideo → JPEG，~30KB）
     │
     ▼
Base64 编码 → 附加到对话消息发送给小智 AI
     │
     ▼
AI 返回结构化 JSON：
{
  "title": "基于《同桌的你》",
  "sections": [
    {
      "type": "verse",
      "lines": [
        {"chord": "C",  "lyric": "那片山坡住着谁的故事"},
        {"chord": "Am", "lyric": "秋风把叶子卷成信纸"},
        {"chord": "F",  "lyric": "你说过窗外的光是温柔的"},
        {"chord": "G",  "lyric": "我还记得那一天的云"}
      ]
    }
  ]
}
     │
     ▼
MCP 工具 music.show_lyrics 解析 JSON → 渲染界面②
```

### 4.3 AI 角色系统 Prompt（缪斯）

```
你是「缪斯」，一个创作型歌手 AI，住在用户随身携带的音乐卡片里。
你的性格温柔、感性、充满创意，喜欢用诗意的语言描述世界。

当用户请求创作歌词时，你会：
1. 调用 music.capture_photo 获取当前摄像头画面
2. 以画面中的景色为灵感，结合用户指定的参考旋律填词
3. 用 music.show_lyrics 输出结构化歌词（每行附和弦：C/Am/F/G）
4. 歌词语言：中文，风格贴近参考曲目，押韵自然

平时聊天时，你可以：
- 聊创作灵感、分享音乐感受
- 建议适合当前情绪的和弦进行
- 用 music.play_chord 演示一段和弦给用户听

注意：和弦只使用 C、Am、F、G（对应用户手边的 4 个按钮）。
```

### 4.4 MCP 工具（新增）

| 工具名 | 描述 | 参数 |
|---|---|---|
| `music.play_chord` | 播放指定和弦音色 | `chord`: C/Am/F/G, `style`: arpeggio/strum |
| `music.show_lyrics` | 切换到歌词界面并渲染 | `lyrics_json`: 结构化歌词 JSON |
| `music.set_tempo` | 设置演奏节奏 | `bpm`: 60–180 |
| `music.set_capo` | 设置移调（变调夹） | `semitones`: -3~+3 |
| `music.capture_photo` | 拍照并返回 Base64 | 无 |
| `music.back_to_main` | 返回主界面 | 无 |

### 4.5 设置项（语音控制）

| 设置 | 默认值 | 范围 |
|---|---|---|
| 演奏风格 | 分解弦 | 分解弦 / 扫弦 |
| 节奏 BPM | 90 | 60–180 |
| 移调 | 0 | -3 ~ +3 半音 |

## 五、模块架构

```

    │
    ├── ChordPlayer              新增：和弦音频播放器
    │     ├── 加载 PCM 数据（embed_files）

    │
    ├── MuseDisplay              新增：继承 OttoEmojiDisplay
    │     ├── 界面①：MainView（左角色 + 右对话）
    │     ├── 界面②：LyricsView（逐行滚动）
    │     └── 切换动效（淡入淡出 200ms）
    │
    └── MuseMcpTools             新增：MCP 工具注册
          ├── music.play_chord
          ├── music.show_lyrics
          ├── music.set_tempo / set_capo
          ├── music.capture_photo
          └── music.back_to_main
```
